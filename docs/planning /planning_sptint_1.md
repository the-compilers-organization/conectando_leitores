# 🟢 Sprint 1 – Planejamento e Definição do Backlog

------------------------------------------------------------------------

## 1️⃣ Objetivo da Sprint

Definir a estrutura inicial do projeto Conectando Leitores, organizando os Epics e criando as histórias de usuário para compor o Product Backlog, garantindo clareza das funcionalidades e preparação para o desenvolvimento.

------------------------------------------------------------------------

## 2️⃣ Backlog da Sprint

As seguintes histórias de usuário foram selecionadas do Product Backlog:

### 🔹 US00.1 – Estruturar Epics do Sistema

## Epic

Relacionado ao épico: Backlog

------------------------------------------------------------------------

## História de Usuário

Como **equipe de desenvolvimento**\
quero **definir os Epics do sistema**\
para **organizar as funcionalidades principais do projeto**.

------------------------------------------------------------------------

## Descrição

Identificar e estruturar os Epics que representarão as funcionalidades principais do sistema.

------------------------------------------------------------------------

## Critérios de Aceitação

- [ ] Epics principais definidos
- [ ] Epics documentados no GitHub Projects
- [ ] Epics organizados por funcionalidades

------------------------------------------------------------------------

## Definition of Done

- [ ] Implementado
- [ ] Testado
- [ ] Documentado

------------------------------------------------------------------------

### 🔹 US00.2 – Criar Histórias de Usuário
