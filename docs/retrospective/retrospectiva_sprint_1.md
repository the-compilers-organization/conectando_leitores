# 🟢 Retrospectiva da Sprint 1 – Planejamento e Definição do Backlog

📅 **Período:** 01/03/2026 a 14/03/2026  
⏱️ **Duração:** 2 semanas  
🎯 **Foco da Sprint:** Planejamento inicial e organização do backlog  

---

## 1️⃣ Objetivo da Sprint

A Sprint 1 teve como objetivo estruturar o projeto **Conectando Leitores**, definindo:

- O domínio do sistema  
- Os Epics principais  
- As histórias de usuário  
- A organização do Product Backlog  

Essa etapa foi essencial para garantir uma base sólida para o desenvolvimento.

---

## 2️⃣ O que funcionou bem ✅

- ✔ Boa comunicação entre os membros da equipe  
- ✔ Participação ativa de todos os integrantes  
- ✔ Organização eficiente das tarefas  
- ✔ Definição clara do domínio do sistema  
- ✔ Estruturação dos Epics de forma organizada  
- ✔ Criação das histórias de usuário com critérios de aceitação  
- ✔ Uso do GitHub Projects para organizar o backlog  

---

## 3️⃣ O que não funcionou bem ⚠️

- ❌ Inexperiência inicial com metodologia Scrum  
- ❌ Dificuldade no uso do GitHub Projects  
- ❌ Desafios na escrita das histórias de usuário  
- ❌ Dúvidas na priorização das funcionalidades  
- ❌ Tempo maior que o esperado para organização inicial  

---

## 4️⃣ O que pode ser melhorado 🔧

- 🔹 Aprimorar o entendimento sobre metodologias ágeis  
- 🔹 Melhorar a escrita de histórias de usuário (formato e clareza)  
- 🔹 Definir critérios de aceitação mais detalhados (BDD)  
- 🔹 Melhorar a priorização do backlog  
- 🔹 Organizar melhor o fluxo de trabalho no board  

---

## 5️⃣ Plano de Ação para a Próxima Sprint 🚀

- 📌 Realizar estudo em grupo sobre Scrum e boas práticas  
- 📌 Refinar histórias de usuário antes da implementação  
- 📌 Atualizar o GitHub Projects com maior frequência  
- 📌 Melhorar a divisão de tarefas entre os membros  
- 📌 Revisar backlog antes do início da Sprint 2  

---

## 6️⃣ Lições Aprendidas 📚

- 💡 O planejamento é fundamental para o sucesso do projeto  
- 💡 Uma boa definição de backlog reduz problemas futuros  
- 💡 Histórias de usuário bem escritas facilitam o desenvolvimento  
- 💡 Ferramentas como GitHub Projects exigem prática  
- 💡 A colaboração da equipe é essencial  

---

## 7️⃣ Resultado Geral da Sprint 📊

- ✔ Epics definidos e organizados  
- ✔ Histórias de usuário criadas  
- ✔ Backlog estruturado  
- ✔ Projeto preparado para início do desenvolvimento  

---

## 📌 Conclusão

A Sprint 1 foi fundamental para estabelecer a base do projeto.  
Apesar das dificuldades iniciais com ferramentas e metodologias, a equipe conseguiu estruturar o backlog de forma consistente, permitindo avançar para as próximas etapas com mais segurança e organização.
